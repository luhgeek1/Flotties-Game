# Flottie's Game

Браузерная интеллектуальная викторина для **3 игроков за одним компьютером**:
**2 основных раунда + финал**, с поддержкой обычных вопросов, **«Кота в мешке»** и **«Аукциона»**.

Проект ориентирован на **десктоп**.

## Быстрый старт

### Требования

- Node.js **LTS**

### Запуск (как проверяют по ТЗ)

```bash
npm ci
npm start
```

После запуска приложение доступно по адресу:
`http://localhost:4173/`

`npm start` поднимает `vite preview` на порту `4173` и отдает готовую статику из папки `public`.

### Сборка статики

```bash
npm run build
```

Сборка идет в папку `public`.

Техническая конфигурация:

- `vite.config.ts`: `build.outDir = "public"`
- `vite.config.ts`: `base = "./"`
- `vite.config.ts`: `publicDir = false`

## Как играть

### Основной флоу

1. **Онбординг (первый запуск)**
   - короткое знакомство с игрой и механиками от ведущего
   - демонстрация базового сценария
   - возможность пропустить и начать подготовку
2. **Setup (подготовка партии)**
   - выбор/создание профилей игроков
   - выбор пака вопросов
   - переходы в Магазин и Историю
3. **Игра**
   - Раунд 1
   - Раунд 2 (с промежуточными итогами)
4. **Финал**
   - подтверждение готовности
   - тема финала
   - секретные ставки по очереди
   - ответы по очереди
   - итоговые результаты

Большинство игровых сценариев реализовано через модальные окна
(вопрос, аукцион, кот в мешке и т.д.).

Состояние игры сохраняется и корректно восстанавливается после перезагрузки страницы.

## Управление

### Клавиши игроков (по умолчанию)

- Игрок 1: `A`
- Игрок 2: `Space`
- Игрок 3: `L`

### Действия

| Действие | Как выполнить |
| --- | --- |
| Выбрать вопрос на табло | Клик по ячейке |
| Захватить право ответа | Нажать клавишу своего игрока |
| Ввести и отправить ответ | Поле ввода + кнопка «Ответить» или `Enter` |

## Обычный вопрос: таймеры и попытки

### Стадия 1: Reading (перехват права ответа)

- После открытия вопроса запускается таймер **30 секунд**.
- Пока таймер идет, любой игрок может нажать свою клавишу и захватить право ответа.
- После перехвата таймер паузится.

### Стадия 2: Answering (ввод ответа)

- На ввод ответа дается отдельный таймер: **15 секунд**.
- Через 7 секунд появляется подсказка от ведущего «Отвечай скорее».
- Через 15 секунд (если ответа нет) ответ автоматически считается неверным.

### Начисление очков

- Верный ответ: `+стоимость вопроса`.
- Неверный ответ / таймаут ответа: `-стоимость вопроса`.
- Один и тот же игрок не может отвечать повторно на один и тот же вопрос.
- После ошибки игрок исключается из повторного перехвата на этом вопросе.

### После ошибки

- Игра возвращается в стадию перехвата.
- Таймер перехвата продолжается с момента паузы.
- Если перехватный таймер полностью истек, вопрос закрывается и становится недоступным.

## Спец-вопросы

Спец-вопросы выпадают неожиданно: игрок заранее не знает, какая клетка окажется специальной. Каждый раз это новая клетка. 

### Кот в мешке

1. Игрок, открывший вопрос, выбирает адресата (одного из оставшихся игроков).
2. Адресат выбирает ставку:
   - Раунд 1: `100` или `500`
   - Раунд 2: `200` или `1000`
3. Отвечает только адресат.
4. Очки:
   - верный ответ: `+ставка`
   - неверный ответ: `-ставка`
5. Право выбора следующего вопроса получает адресат.

### Аукцион

- Допускаются игроки, у кого `score >= номинал вопроса`.
- Если никто не проходит по номиналу, вопрос играется как обычный.
- Если проходит 1-2 игрока, показывается предупреждение об ограниченном аукционе.

Правила ставок:

- Если ставок еще нет, минимум = номинал.
- Если лидер есть и он не ва-банк, минимум = `max(номинал, ставка лидера + 1)`.
- Если лидер ушел ва-банк, перебить можно только ва-банком:
  - ставка должна быть ровно собственным балансом;
  - и только если баланс больше ставки лидера.
- Максимум ставки всегда ограничен текущим балансом игрока.

Пас:

- Пас окончательный: игрок выбывает из торгов.
- Если у игрока была ставка, при пасе она сбрасывается.

Завершение аукциона:

- Когда остается 1 активный игрок, торги завершаются.
- Победитель отвечает на вопрос по своей финальной ставке (или по номиналу, если ставок не было).
- Очки:
  - верный ответ: `+ставка`
  - неверный ответ: `-ставка`

Также доступны быстрые действия: минимальная ставка и ва-банк.

## Финал

В финале участвуют только игроки с положительным ненулевым счетом после второго раунда.
Если таких игроков нет, игра завершается без победителя.

Флоу финала:

1. Подтверждение готовности.
2. Показ темы финала.
3. Секретные ставки по очереди:
   - видно, кто делает ставку;
   - выводится плашка «остальные отворачиваются»;
   - есть кнопка «Я готов» перед вводом;
   - перед вводом показывается короткая анимированная подсказка ведущего.
4. Секретные ответы по очереди:
   - аналогичная механика «Я готов»;
   - на ответ дается **60 секунд**;
   - таймер стартует сразу после подсказки ведущего.
5. После всех ответов раскрываются результаты и обновляются очки.

## Dev-mode (админ-режим)

Включается тумблером **Admin** в правом верхнем углу (доступен на всех экранах).

Что дает режим:

- Подсветка спец-вопросов на табло:
  - красный: «Кот в мешке»
  - фиолетовый: «Аукцион»
- В модалке ответа автоподстановка правильного ответа (включая финал).
- Кнопка «Закончить раунд» для ускоренной проверки сценариев.
- В магазине админ получает 1000 монет для тестирования покупок.

## Оптимизация загрузки ассетов

Для ускорения загрузки на GitLab Pages используется предзагрузка изображений.

Файлы:

- утилиты: `src/shared/lib/imagePreload.ts`
- preload-план по роутам: `src/app/router/model/routeImagePreload.ts`
- точка запуска: `src/app/router/model/useRouteImagePreload.ts`

Подход:

- критичные ассеты текущего этапа грузятся сразу (`fetchPriority: "high"`);
- остальные догружаются в idle через `requestIdleCallback` (или fallback на `setTimeout`).

## Архитектура и стек

### Stack

- React + TypeScript + Vite
- Tailwind CSS + shadcn/ui
- Motion
- Jotai

Alias: `@ -> ./src` (см. `vite.config.ts`).

### Архитектура

Используется подход **FSD (Feature-Sliced Design)**:

- `src/app` — роутинг, глобальные настройки, app-level логика
- `src/pages` — страницы (`setup`, `game`, `final*`, `history`, `shop`)
- `src/widgets` — крупные композиционные блоки
- `src/features` — игровые сценарии (question flow, auction, cat-in-bag, onboarding, shop, history, admin)
- `src/entities` — доменные сущности
- `src/shared` — переиспользуемые UI/утилиты/assets/storage

### Роутинг

Навигация реализована в `src/app/router` через внутренний route-state и hash-url (`/#/...`).
Route guard ограничивает переходы в зависимости от этапа игры и состояния.
Hash-формат URL нужен для GitLab Pages: при обновлении страницы на вложенных путях приложение не получает `404`.
Реализация находится в `src/app/router/model/routeLocation.ts` (`getLocationPath`, `toHashUrl`, `replaceHashPath`, `pushHashPath`).

| Путь | Экран | Назначение |
| --- | --- | --- |
| `/` | Setup | Выбор игроков, пака, вход в игру |
| `/shop` | Shop | Магазин/косметика |
| `/history` | History | История игр |
| `/game` | Game Round 1 | Первый раунд |
| `/game2r` | Game Round 2 | Второй раунд |
| `/final-prepairing` | FinalPrepairing | Подготовка к финалу |
| `/final-start-theme` | FinalStartTheme | Объявление темы финала |
| `/final-close-eyes` | FinalCloseEyes | Подготовка перед ставками/ответами |
| `/final-bid` | FinalBid | Секретные ставки |
| `/final-close-eyes-question` | FinalCloseEyes | Подготовка перед ответами |
| `/final-question` | FinalQuestion | Ответы на финальный вопрос |
| `/final-results` | FinalResults | Итоги и победитель |

### Взаимодействие модулей

- `useAppNavigation` управляет маршрутом, синхронизацией history и route-guard.
- `useGamePageModel` собирает игровую страницу из `game-session`, `player-pick`, `question-flow`, `special-interactions`.
- Состояние хранится в Jotai-атомах с persistent-слоем в `localStorage`.
